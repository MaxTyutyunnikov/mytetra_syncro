В процессе отмены предыдущих (плохих) изменений, вы можете обнаружить, что было бы полезно проверить более старую версию файла. Делается это командой git checkout. Обычно, git checkout используется для того, что бы сменить текущую ветку. Однако эта команда работает совершенно по-другому, если в качестве аргумента получает путь к файлу:



$ git checkout HEAD^ path/to/file


Вышеприведенная комазаменяет содержимое указанного файла, и из коммита HEAD^, а также обновляет соответственно индекс. Команда не меняет текущую ветвь. Если вы просто хотите взглянуть на старую версию файла без изменения рабочего каталога, вы можете сделать git-show(1):

$ git show HEAD^:path/to/file


который будут отображён с учетом версии файла.
