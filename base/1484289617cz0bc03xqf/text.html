Для того, чтобы многократно не включать модуль в другие модули (хотя это не возброняется, просто нежелательно), нужно помнить следующее:



Предположим, есть три модуля:


Main.py

Config.py

ConfigClass.py


В файле Config.py создается экземпляр класса ConfigClass, и наполняется значениями. Выглядит файл Config.py так:


from ConfigClass import *


config=ConfigClass()


config.addItem("MyName1", "MyValue1")

config.addItem("MyName2", "MyValue2")


А в файле Main.py нужно импортировать этот экземпляр (с именем config) и нужно уметь работать с методами класса ConfigClass. Вопрос. Нужно ли в Main.py импортить

прописано:


from Config import *
