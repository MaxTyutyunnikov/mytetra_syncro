Чтобы узнать системную кодировку в Windows, надо понимать, что в Qt нет готовых методов. Поэтому придется делать все с помощью функции WinAPI GetACP() и условной компиляцией.


Код будет выглядеть примерно так:



#ifdef Q_OS_WIN32

#include "windows.h"

#endif


using namespace std;


QString getSystemCodepage()

{

#ifdef Q_OS_WIN32

return "CP "+QString::number( GetACP() );

#endif


#ifdef Q_OS_LINUX

return "UTF-8");

#endif


return "";

}



А использовать эту функцию можно примерно так:



QString codePage=getSystemCodepage();

qDebug() << "System code page: " << codePage;

if(codePage.size()>0) {

QTextCodec::setCodecForLocale(QTextCodec::codecForName(codePage.toLatin1()));

}



После выполнения такого кода в русскоязычной Windows, будет установлена кодировка локали CP1251. Кодировка локали для Windows - это та кодировка, в которой записываются имена файлов и директорий. Однако следует помнить, что кодировка консоли отличается, и в русскоязычной Windows для консоли принята кодировка CP 866.


О том, как можно одновременно работать с двумя кодировками, рассказывается в статье Как в Qt работать с русскими именами файлов и директорий под Windows.
