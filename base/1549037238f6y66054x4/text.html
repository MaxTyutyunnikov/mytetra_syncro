Стандарт BIP


Есть несколько стандартов таких фраз-ключей:


BIP39 Mnemonic code for generating deterministic keys

Read more at the official BIP39 spec

BIP32 Hierarchical Deterministic Wallets

Read more at the official BIP32 spec

See the demo at bip32.org

BIP44 Multi-Account Hierarchy for Deterministic Wallets

Read more at the official BIP44 spec

BIP49 Derivation scheme for P2WPKH-nested-in-P2SH based accounts

Read more at the official BIP49 spec


Пояснение на русском: существуют BIP 32, BIP 39, BIP 44, и еще BIP 49. Самый распространенный - это BIP 44 (12 слов). BIP 39 - это то же самое, только есть возможность выбирать количество слов самостоятельно. Но BIP 39 из 12 слов это то же самое что BIP 44. Если вы забэкапите кошелек именно в этом виде, то вы гарантированно восстановите его почти в любом другом месте через длительное время. Гораздо меньше кошельков поддерживают BIP 39 с 15-тью и более словами. BIP 32 (13 слов) это устаревший формат, его использовать не желательно.



Кошельки


Copay - поддерживает только биткоин, генерирует и восстанавливается по BIP 44/39.

Coinomi - поддерживает очень много валют, восстанавливается по BIP 44/39, но генерирует только BIP 39 из 18-ти слов.

Electrum - поддерживает только биткоин, восстанавливается из BIP44 или BIP 32, но генерирует только BIP 32.

Electrum для DASH и LTC - поддерживают работу только с BIP 32.

Mobile DASH для iOS - поддерживает только DASH, генерирует и восстанавливается по BIP 44. Версия для Android (пока) BIP 44 не поддерживает.


Инструмент для просмотра ключей через мнемотическую фразу: https://iancoleman.io/bip39/ (подходит только для скомпроментированных кошельков).

Он же для выкачивания и использования в оффлайне: https://iancoleman.io/bip39/ . Для сборки итогового html-файла используется скрипт на Питоне compile.py, в результате должен получиться файл bip39-standalone.html.
