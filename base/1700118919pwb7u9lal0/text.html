Бывает так, что при выполнении команды make install необходимо сделать копирование некоторых файлов проекта в какие-либо директории. Сам Makefile, естественно, генерируется через qmake (cmake здесь не рассматривается). Вопрос в том, как в синтаксисе *.pro-файла написать директивы копирования заданных файлов.


Делается это следующим образом: вначале задается именованный список файлов, которые подлежат копированию. Выполняется это через директиву *.files, где вместо звездочки можно писать любое имя, которое прийдет в голову:


Вот как выглядит именованный список с именем features, состоящий из одного файла:



features.files = $${PWD}/features/loginWindow.prf



Вот как выглядит именованный список с именем installators, состоящий из нескольких файлов:



installators.files = \

$${PWD}/scripts/install_win.bat \

$${PWD}/scripts/install_linux.sh \

$${PWD}/scripts/install_macos.sh



Переменная $${PWD} обозначает место, где находится *.pro-файл (или Makefile?), и в общем случае его можно даже не указывать.


Директория назначения, куда будут скопированны файлы, задается через директиву *.path, где вместо звездочки пишется имя именованного списка файлов, показанного выше.
