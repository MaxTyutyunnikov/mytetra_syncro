Настройка количества монтирований, после которых происходит проверка ФС


У файловых систем ext2, ext3,ext4 есть параметры отвечающие через

какое время (количество монтирований) будет выполнена их проверка. По-

моему, эту опцию следует обязательно отключать на серверах, иначе можно

очень долго волновать почему сервер не вышел из перезагрузки. За это

отвечает два параметра:



  1. Check interval — через какое время будет произведена следующая проверка.
  2. Maximum mount count — через сколько монтирований будет выполнена проверка.



Просмотреть текущие значения этих параметров можно с помощью команды tune2fs:


~$ sudo tune2fs -l /dev/md0 |grep "ount count"


Mount count: 29


Maximum mount count: -1



~$ sudo tune2fs -l /dev/md0 |grep "Check interval"


Check interval: 0 (<none>)



Отключаем эти проверки для раздела /dev/sdb2:



$sudo tune2fs -i=0 /dev/sdb2


$sudo tune2fs -c 0 /dev/sdb2
