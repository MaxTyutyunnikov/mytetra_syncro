Отдельно устанавливать Bootstrap CSS и Vue.Js в Laravel нет необходимости. Они идут в комплекте по-умолчанию.


Например, предположим что Laravel установлен через Composer. То есть, в корневом каталоге веб-сервера /var/www/myPrettySite были выполнены команды:


composer global require "laravel/installer"

composer create-project --prefer-dist laravel/laravel site



В результате создался подкаталог site, внутри которого есть файлы composer.json и package.json.

Файл package.json - это файл, в котором записан конфиг, который применяет в своей работе пакетный менеджер npm. Если заглянуть в него, то можно увидеть, что и Bootstrap и Vue.Js есть в зависимостях. И даже все файлы компонентов Bootstrap и Vue уже есть в подкаталоге site.


Однако эти файлы были установлены при разворачивании сайта через composer, и о них не знает пакетный менеджер npm, потому что он еще не был запущен. Для запуска npm и для первичной конфигурации надо выполнить команду:



npm install



Эта команда проверит файл package.json и установит все пакеты, которые прописаны в данном файле. А так как все файлы из пакетов уже были развернуты через composer, то, по-сути, ничего не произойдет. Просто проверится что файлы уже есть, если, конечно, не изменились версии пакетов.
