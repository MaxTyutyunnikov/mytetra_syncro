Создание и использование интерфейсов




Интерфейсы во многом схожи с абстрактными классами. Интерфейс - это шаблон, который задает поведение одного или более классов.


Вот основные отличия между интерфейсами и абстрактными классами:

Ни один метод не может быть описан в интерфейсе. Они все абстрактны. В абстрактном классе могут быть и не абстрактные методы.

Интерфейс не может содержать полей - только методы.

Класс имплементирует интерфейс, и класс наследует или расширяет другой класс.

Класс может имплементировать несколько интерфейсов одновременно. Этот же класс может наследовать другой класс. Но у дочернего класса может быть только один супер-класс (абстрактный или нет).


Как и абстрактный класс, интерфейс объявляет несколько методов, которые должны быть реализованы в любом классе, который имплементирует данный интерфейс. Синтаксис выглядит так:1 interface MyInterface {

2 public function aMethod();

3 public function anotherMethod();

4 }



Чтобы создать класс, который имплементирует тот или иной интерфейс, напишите так:01 class MyClass implements MyInterface {

02

03 public function aMethod() {

04 // (имплементация метода)

05 }

06

07 public function anotherMethod() {

08 // (имплементация метода)

09 }

10

11 }



Интерфейсы полезны в случаях, когда нужно создать несколько несвязанных между собой классов, у которых будет общая функциональность.


Например, веб-форум может содержать класс Member для участников форума и класс Topic для тем, создаваемых участниками форума. В отношении наследования, эти классы скорее всего не будут зависеть друг от друга, так как они выполняют совершенно разные функции.


Тем не менее, давайте предположим, что нам нужно будет доставать их и записывать в базу данных как объекты класса Member, так и объекты Topic. Для этого мы создадим интерфейс Persistable, в котором будут методы для сохранения объектов в БД и извлечения их оттуда:1 interface Persistable {

2

3 public function save();

4 public function load();

5 public function delete();

6

7 }



Теперь давайте создадим класс Member и имплементируем для него интерфейс Persistable. Это значит, что в интерфейсе должны быть методы save(), load() и delete():01 class Member implements Persistable {

02

03 private $username;

04 private $location;

05 private $homepage;

06

07 public function __construct( $username, $location, $homepage ) {

08 $this->username = $username;

09 $this->location = $location;

10 $this->homepage = $homepage;

11 }

12

13 public function getUsername() {

14 return $this->username;

15 }

16

17 public function getLocation() {

18 return $this->location;

19 }

20

21 public function getHomepage() {

22 return $this->homepage;

23 }

24

25 public function save() {

26 echo "Saving member to database<br>";

27 }

28

29 public function load() {

30 echo "Loading member from database<br>";

31 }

32

33 public function delete () {

34 echo "Deleting member from database<br>";

35 }

36

37 }



Наш класс Topic также будет имплементировать данный интерфейс, поэтому в нем тоже должны быть эти три метода:01 class Topic implements Persistable {

02

03 private $subject;

04 private $author;

05 private $createdTime;

06

07 public function __construct( $subject, $author ) {

08 $this->subject = $subject;

09 $this->author = $author;

10 $this->createdTime = time();

11 }

12

13 public function showHeader() {

14 $createdTimeString = date( 'l jS M Y h:i:s A', $this->createdTime );

15 $authorName = $this->author->getUsername();

16 echo "$this->subject (created on $createdTimeString by $authorName)<br>";

17 }

18

19 public function save() {

20 echo "Saving topic to database<br>";

21 }

22

23 public function load() {

24 echo "Loading topic from database<br>";

25 }

26

27 public function delete () {

28 echo "Deleting topic from database<br>";

29 }

30

31 }



На заметку: так как у нас форум - воображаемый, вместо взаимодействия с базой данных в методах save(), load() и delete() просто выводятся сообщения.


Мы также добавили несколько дополнительных методов, например, Member::getUsername() для получения имени пользователя, и Topic::showHeader() для отображения названия темы, имени автора и времени создания темы.


Теперь можем создать объекты классов Member и Topic, а затем вызвать их методы getUsername() и showHeader(). Более того, зная, что эти классы имплементируют интерфейс Persistable, мы можем вызывать такие методы, как save(), load() или delete():1 $aMember = new Member( "fred", "Chicago", "http://example.com/" );

2 echo $aMember->getUsername() . " lives in " . $aMember->getLocation() ."<br>";

3 $aMember->save();

4

5 $aTopic = new Topic( "Teddy Bears are Great", $aMember );

6 $aTopic->showHeader();

7 $aTopic->save();



На странице отобразится:1 fred lives in Chicago

2 Saving member to database

3 Teddy Bears are Great (created on Wednesday 25th May 2011 02:19:14 AM by fred)

4 Saving topic to database



Как видите, интерфейсы могут соединить классы, которые никак не относятся друг к другу, для определенных целей, например, записи в базу данных объектов класса. Не забудьте, что один класс может имплементировать несколько интерфейсов:1 class MyClass implements anInterface, anotherInterface {

2 ...

3 }



На заметку: интерфейсы - это мощное свойство ООП, и о них можно еще много чего сказать. Узнать о них больше можно в PHP документации.
